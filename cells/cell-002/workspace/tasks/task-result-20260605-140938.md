    任務理解  
本任務需設計商品查詢 API，支援依名稱、分類、價格區間查詢，並回傳商品清單。設計需符合 Hexagonal Architecture，確保關注點分離與可擴充性。

執行結果  
- API Endpoint：`GET /products`
- 查詢參數：
  - `name`（商品名稱，模糊查詢）
  - `category`（商品分類）
  - `minPrice`、`maxPrice`（價格區間）
  - `page`、`size`（分頁）
- 回傳格式：
  - `products`: 商品陣列（含 id、name、category、price、description）
  - `total`: 總筆數
- 架構規劃：
  - Controller：接收查詢參數
  - Service：組合查詢條件
  - Repository：查詢資料庫
  - Domain：Product Entity

3. 實作 Service 層查詢邏輯。
