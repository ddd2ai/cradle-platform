    任務理解  
本任務需設計並實作商品查詢 API，支援依條件查詢商品，服務 product、catalog、search 等模組。實作需符合 Hexagonal 架構與 Spring Boot 實作規範，並考量可擴充性與維護性。

執行結果  
- 已確認需求屬於電商系統核心功能，影響多個模組。
- 初步規劃 API 介面：GET /products，支援依名稱、分類、價格等條件查詢。
- 預計分層：Controller（API 入口）、Application（查詢邏輯）、Domain（商品物件）、Infrastructure（資料存取）。
- 技術棧：Java 21、Spring Boot、JPA（連接 MariaDB）。

下一步建議  
1. 設計商品 Entity 與 Repository 介面，定義查詢條件。
2. 實作 Application Service，處理查詢邏輯。
3. 建立 Controller，串接查詢服務並回傳結果。
4. 撰寫單元測試與整合測試，確保查詢正確性。
5. 文件化 API 規格，方便前端與其他模組串接。
