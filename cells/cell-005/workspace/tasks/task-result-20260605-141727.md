    ## Result

    任務理解  
本任務需設計一組訂單查詢 API，支援依訂單編號、會員 ID、日期區間查詢，並回傳訂單狀態等資訊。設計需符合 Hexagonal Architecture，確保結構清晰、易於維護與擴充。

執行結果  
- API 路徑建議：GET /orders
- 查詢參數：
  - orderNo（訂單編號，精確查詢）
  - memberId（會員 ID，精確查詢）
  - startDate/endDate（日期區間，格式 yyyy-MM-dd）
- 回傳格式：List<OrderDto>，每筆包含 orderNo、memberId、orderDate、status、totalAmount 等欄位
- 架構分層：
  - Controller：接收查詢請求與參數
  - Application Service：組合查詢條件，調用查詢用例
  - Domain：定義訂單實體與查詢規則
  - Repository：負責資料查詢，與 MariaDB 整合

下一步建議  
1. 定義 Order Entity、OrderDto、OrderQueryCriteria 等核心物件。
2. 設計 Repository 介面與查詢實作（JPA/QueryDSL）。
3. 撰寫 Controller 與 Application Service，串接查詢流程。
4. 撰寫單元測試與 API 文件，確保設計正確且易於理解。
