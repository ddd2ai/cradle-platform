# Task Result

    ## Task

    商品查詢 API 效能優化

    ## Task ID

    task-20260605-141207

    ## Result

    任務理解  
本任務針對商品查詢 API 進行效能優化，重點在於：  
- 實作分頁查詢，避免一次回傳大量資料  
- 設計適當資料庫索引，加速查詢  
- 優化關鍵字搜尋（如 LIKE、全文檢索）  
- 評估快取策略，減少重複查詢壓力  
目標是讓 API 在高流量下仍能穩定快速回應。

執行結果  
- 分頁查詢：API 增加 page/size 參數，Repository 層採用分頁查詢（如 JPA Pageable）。
- 索引設計：針對 name、category、price 欄位建立複合索引，提升查詢效率。
- 關鍵字搜尋：優先採用資料庫 LIKE 查詢，若效能不足可評估全文檢索（如 MariaDB FULLTEXT）。
- 快取策略：可於 Service 層導入本地快取（如 Caffeine），或評估 Redis 等分散式快取，針對熱門查詢結果快取。

下一步建議  
1. 針對實際查詢流量進行壓力測試，觀察瓶頸。
2. 持續監控查詢效能，根據指標調整索引與快取策略。
3. 若資料量持續成長，考慮分表或搜尋引擎（如 ElasticSearch）。
4. 補充效能優化相關文件與測試案例，確保團隊理解設計原則。
