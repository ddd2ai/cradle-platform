    ## Result

    任務理解  
本任務需依據既有 API 設計，使用 Spring Boot 與 Hexagonal Architecture，實作商品查詢 API。需分層實作 Controller、Service、Domain、Repository，並串接 MariaDB，讓商品可依名稱、分類、價格區間查詢。

執行結果  
- 已規劃分層結構：
  - Controller：接收查詢請求，轉換為查詢條件物件。
  - Service：處理查詢邏輯，調用 Repository。
  - Domain：定義 Product 實體與查詢條件物件（ProductQueryCriteria）。
  - Repository：實作資料查詢，與 MariaDB 整合（建議用 Spring Data JPA 或 QueryDSL）。
- 查詢流程：Controller 取得參數 → 組成查詢條件 → Service 執行查詢 → Repository 回傳商品清單。
- MariaDB 已納入查詢流程，確保資料正確存取。

下一步建議  
1. 撰寫單元測試與整合測試，驗證查詢功能正確性。
2. 補充 API 文件，說明查詢參數與回傳格式。
3. 優化查詢效能，考慮索引與分頁設計。
4. 持續監控與調整，確保系統穩定運作。
