    ### Result Summary
    任務理解  
本任務需設計一組管理報表查詢 API，支援管理端查詢各類營運數據（如訂單、會員、銷售統計等）。設計需符合 Hexagonal Architecture，確保分層清楚、易於擴充，並能因應未來報表類型變動。

執行結果  
- API 路徑建議：GET /reports/{type}  
- 查詢參數：type（報表類型）、startDate/endDate、其他動態條件（如 memberId、category）
- 回傳格式：List<ReportDto>，欄位依報表類型動態調整
- 架構分層：  
  - Controller：接收查詢請求與參數  
  - Application Service：組合查詢條件，調用查詢用例  
  - Domain：定義報表查詢規則與資料結構  
  - Repository：負責資料查詢，與 MariaDB 整合

下一步建議  
1. 定義 ReportDto、ReportQueryCriteria 等核心物件，支援多種報表型態。
2. 設計 Repository 介面與查詢實作，考慮動態 SQL 或查詢模板。
3. 撰寫 Controller 與 Application Service，串接查詢流程。
4. 撰寫單元測試與 API 文件，確保設計正確且易於理解。
5. 評估報表查詢效能，必要時導入快取或批次預算。
