    ## Task Experience

    ### Task
    設計 Payment Gateway 架構與 API

    ### Source
    metabolism

    ### Result Summary
    任務理解  
本任務需設計一套可擴充的 Payment Gateway 架構，支援信用卡、轉帳付款與退款，並規劃 REST API 介面，且能自動產生 OpenAPI 文件。需考量 Hexagonal 架構、未來支付方式擴充性，以及與現有電商系統整合。

執行結果  
- 架構建議：採用 Hexagonal Architecture，將付款邏輯（信用卡、轉帳、退款）封裝於 Application 層，外部介接 REST API，Domain 層抽象 Payment/Refund 介面，Infrastructure 層實作各支付通道。
- API 路徑建議：
  - `POST /payments/credit-card`
  - `POST /payments/bank-transfer`
  - `POST /refunds`
- 支援 OpenAPI（Swagger）自動產生，建議整合 Springdoc 或 Swagger 套件。
- 介面設計可用 DTO，明確定義輸入/回傳格式，便於維護與擴充。

下一步建議  
1. 詳細設計 Payment/Refund Entity 與資料表結構。
2. 撰寫 Use Case 與 Service 介面，定義付款、退款流程。
3. 實作 Repository 串接 MariaDB。
4. 製作 Controller 串接 Use Case，並整合 OpenAPI 文件產生。
5. 撰寫單元測試與 API 驗證，確保流程正確性與 API 文件完整。
