    ## Task

    API 認證與授權檢查

    ## Task ID

    ## Result

    任務理解  
本任務需全面檢查現有 API，補強 Authentication（身份驗證）與 Authorization（權限驗證）機制，避免未授權存取與資安風險。需考量現有架構（Spring Boot、Hexagonal），並確保所有敏感資源皆有適當防護。

執行結果  
- 已盤點現有 API，發現部分查詢與管理介面缺少權限驗證。
- 建議統一導入 Spring Security，採用 JWT 或 OAuth2 作為身份驗證方案。
- 針對不同 API 路徑設計細緻的權限規則（如 ROLE_USER、ROLE_ADMIN）。
- 補強 Controller 層的 @PreAuthorize 或 @Secured 標註，確保授權檢查。

下一步建議  
1. 實作與測試 JWT/OAuth2 驗證流程。
2. 撰寫 API 權限規則文件，明確定義各角色可存取資源。
3. 製作單元測試與資安測試，驗證未授權請求皆被正確阻擋。
4. 持續監控 API 存取日誌，追蹤異常行為。
