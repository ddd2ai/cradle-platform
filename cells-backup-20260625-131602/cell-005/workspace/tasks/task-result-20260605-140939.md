    ## Result

    任務理解  
本任務需設計一組商品查詢 API，支援依商品名稱、分類、價格區間查詢，並回傳商品清單。設計需符合 Hexagonal Architecture，確保結構清晰、易於維護與擴充。

執行結果  
- API 介面建議：  
  - 路徑：GET /products  
  - 參數：name（商品名稱，模糊查詢）、category（分類）、minPrice/maxPrice（價格區間）
  - 範例：/products?name=手機&category=3C&minPrice=1000&maxPrice=20000
- 查詢條件物件（ProductQueryCriteria）：  
  - name: String  
  - category: String  
  - minPrice: BigDecimal  
  - maxPrice: BigDecimal
- 回傳格式：  
  - List<ProductDto>，每筆包含 id、name、category、price、description 等欄位
- 架構分層：  
  - Controller：接收查詢請求與參數  
  - Application Service：組合查詢條件，調用查詢用例  
  - Domain：定義商品實體與查詢規則  
  - Repository：負責資料查詢，與 MariaDB 整合

下一步建議  
1. 定義 Product Entity、ProductDto、ProductQueryCriteria 等核心物件。
2. 設計 Repository 介面與查詢實作（JPA/QueryDSL）。
3. 撰寫 Controller 與 Application Service，串接查詢流程。
4. 撰寫單元測試與 API 文件，確保設計正確且易於理解。
