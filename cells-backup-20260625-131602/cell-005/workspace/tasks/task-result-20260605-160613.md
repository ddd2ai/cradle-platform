    ## Result

    任務理解  
本任務需根據電商系統現有或新需求，設計並產生標準化的 REST API 規格文件。目標是確保 API 介面清晰、一致，便於前後端協作與後續維護，並符合 Hexagonal Architecture 分層原則。

執行結果  
- 需求盤點：以商品查詢、訂單查詢、付款、退款等核心流程為主。
- API 規格範例（以商品查詢為例）：
  - 路徑：GET /products
  - 參數：name（模糊查詢）、category、minPrice、maxPrice、page、size
  - 回傳：List<ProductDto>，欄位包含 id、name、category、price、description
- 採用 OpenAPI 3.0 標準撰寫規格，便於自動產生文件與測試。

下一步建議  
1. 針對所有核心流程（商品、訂單、付款等）補齊 API 規格。
2. 將 OpenAPI 規格文件納入版本控管，並自動產生 Swagger UI。
3. 定期與前後端協作團隊審查 API 規格，確保需求同步與介面一致。
4. 撰寫 API 測試案例，驗證規格正確性。
