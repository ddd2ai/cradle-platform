    ### Task
    設計 Payment Gateway 架構

    ### Source
    metabolism

    ### Result Summary
    任務理解  
本任務需設計 Payment Gateway 架構，支援信用卡、轉帳付款與退款，並提供 REST API。設計需符合 Hexagonal Architecture，確保結構清晰、易於擴充與維護，並能自動產生 OpenAPI 文件。

執行結果  
- 架構分層：
  - Controller：提供付款、退款等 REST API 入口
  - Application Service：處理付款、退款流程協調
  - Domain：定義 Payment、Refund 等核心模型與規則
  - Infrastructure：串接第三方金流（如信用卡、轉帳）、資料庫
- API 介面建議：
  - POST /payments（發起付款，參數：金額、付款方式、訂單編號等）
  - POST /refunds（申請退款，參數：原付款編號、金額、原因等）
- 支援多種支付方式，PaymentMethod 可擴充
- OpenAPI 文件自動產生（建議用 Springdoc/OpenAPI）

下一步建議  
1. 定義 Payment、Refund、PaymentMethod 等核心物件與介面
2. 設計並實作 REST API Controller 與 Application Service
3. 串接第三方金流服務，設計 Infrastructure Adapter
4. 撰寫單元測試、整合測試與 API 文件
