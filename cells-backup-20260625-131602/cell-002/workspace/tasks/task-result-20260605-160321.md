    ## Task ID

    ## Result

    任務理解  
本任務需設計一套協作協議與任務認領（claim）流程，確保多個 Cell 在處理任務時能自動協調、避免重複執行，並能共享成果與更新協作關係。重點在於任務唯一性、ownership 可追蹤、協作後狀態同步。

執行結果  
- 設計任務資料結構，加入 claimedBy、status、result、collaborators 欄位。
- 任務建立時，先查重（比對標題、內容、hash），有重複則拒絕或合併。
- 認領流程：Cell 先 claim，狀態標記為 claimed，其他 Cell 不可重複認領。
- 支援協作模式（collaborators 欄位），允許多 Cell 共同處理並共享結果。
- 任務完成後，結果與參與 Cell 關係自動更新，供後續分工與演化參考。

下一步建議  
1. 將協作協議與認領邏輯整合進任務管理 API  
2. 設計協作狀態查詢與歷程追蹤介面  
3. 定期檢查未認領或重複任務，主動協調分派  
4. 將協作資料納入 DNA/專長演化依據
